# 第二批：主界面重做（左图右文+翻译tab）

## Goal

主界面按已拍板设计重排：空态大图标 dropzone、有图态左图右文对照、原文/译文双 tab、两段式翻译、操作按钮图标化、去嵌套框。Web/桌面共用。

## Requirements

1. **空态上传区**（用户设想）：大边框 dropzone，中央相机 SVG 图标，下方一行小字"拖拽、点击 或 Ctrl+V 粘贴上传"；框体下方按钮排：粘贴、上传、链接、清除（清除在空态禁用）
2. **有图态布局**（决策 #5）：
   - 顶部工具条：粘贴/上传/链接/清除 + 设置入口
   - 左侧：当前页大图（点击放大弹窗保留）；底部缩略图条 + 页码导航（替代现 ←/→）
   - 右侧：识别结果区；左右之间可拖分隔条
   - 移除 `.upload-section.with-image` 300px 压缩方案
3. **两段式翻译**（决策 #4）：
   - 识别请求恒为纯 OCR；`appendTranslateInstruction` 不再拼进识别 prompt
   - 自动翻译开关 + 目标语言选择移到主界面工具条；开启时识别完成后自动发翻译请求，结果存 `translations[index]`
   - 结果区 tab：原文 / 译文（译文 tab 仅在有翻译或翻译中显示）；对任意已识别结果可手动触发翻译
4. **结果区**：去 `markdown-body` 第二层框；复制/纠错换 SVG 图标按钮 + tooltip；"一键纠错"改名"格式纠错"；复制成功用状态反馈（去 querySelector DOM 操作，修 M3）
5. **错误与引导**：识别失败改 toast + 结果区重试按钮，错误文本不再写入结果（M1）；未配 API Key 时引导跳设置（M2，与第三批衔接）；**重试时在 prompt 末尾追加重试提示**（如"请仔细识别图片中的内容，这是第 N 次尝试，之前的识别并不准确"，用户 2026-07-05 补充）
6. **一致性**：JS 布局判定与 CSS 断点统一（修 #8 的 JS/CSS 打架）；桌面窗口窄化不套移动端样式
7. **CSS 拆分**（Q1 前置清理，范围限本批触及的模块）：上传区/结果区/工具条样式拆为组件级 CSS，去除 App.css 中对应重复定义

## Acceptance Criteria

- [ ] 空态/有图态布局与拍板的 ASCII 示意一致，分隔条可拖动并在合理范围钳制
- [ ] 开启自动翻译：识别流式输出原文 → 完成后自动出译文 tab；关闭时不发翻译请求；手动翻译入口可用
- [ ] 翻译失败不影响原文展示，可重试翻译
- [ ] 多图/PDF 多页：缩略图条切换正常，每页独立原文/译文
- [ ] 识别失败显示 toast + 重试按钮，结果区无错误文本残留
- [ ] 结果区仅一层容器框；复制/纠错为图标按钮且键盘可达（aria-label）
- [ ] Web 端与桌面端表现一致（除桌面专属文案）
- [ ] `npm test` 全绿（useOcrSession 翻译流新增测试；App 布局测试更新）
